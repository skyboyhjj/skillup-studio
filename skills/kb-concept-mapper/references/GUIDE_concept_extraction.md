# 概念提取指南

本指南用于指导 AI 从知识库文档中提取概念并分析关联关系。

---

## 一、概念识别标准

### 什么算"概念"？

概念可以是以下任一类型：

| 类型 | 示例 | 判断标准 |
|------|------|---------|
| **方法论** | 双脑循环、GTD、费曼学习法 | 成体系的做事方法/流程 |
| **框架** | SPARK飞轮、SWOT、OKR | 结构化的分析/操作模型 |
| **工具/平台** | Obsidian、ima.copilot、Notion | 具体的软件/工具/平台名称 |
| **技术** | 大模型、RAG、向量数据库 | 技术术语/技术堆栈 |
| **理论** | 认知负荷理论、心流理论 | 学术/科学理论 |
| **人物** | 墨凡、Tiago Forte | 关键人物/作者 |
| **应用场景** | 知识管理、内容创作、课题研究 | 概念的实践领域 |
| **策略/原则** | 检索增强、最小可行知识 | 指导性的行动原则 |

### 什么**不算**概念？

- 非常通用的词汇（"方法""步骤""问题"）
- 文档本身标题（除非该标题代表一个独立的方法论）
- 没有实质内容支撑的无效概念

### 排除原则

- 文档中仅出现 1 次且非核心关键词 → 排除
- 过于宽泛（"学习""工作""生活"）→ 排除
- 用户姓名、无关人名 → 排除

## 二、关系类型及判定标准

AI 根据上下文判断概念间的关系类型：

| 关系 | 标签 | 判断标准 |
|------|------|---------|
| 包含 | `包含` | A 是 B 的组成部分或子集 |
| 驱动/促进 | `驱动` | A 的使用或存在会增强/促进 B |
| 依赖/前提 | `依赖` | A 的实现需要 B 作为前提 |
| 属于 | `属于` | A 是 B 大类下的一个具体实例 |
| 关联 | `关联` | A 和 B 经常同时出现，有模糊联系 |
| 对比/对立 | `对比` | A 和 B 是同一维度的不同方向 |
| 演进/发展 | `演进` | A 发展或演变为 B |
| 应用 | `应用` | A 方法/理论在 B 场景中应用 |
| 支撑/依据 | `支撑` | A 为 B 提供理论/数据支撑 |

**兜底规则：** 当 AI 无法确定具体关系类型时，使用 `关联` 作为默认关系。

## 三、权重赋值规则

### 节点权重（0~1）

| 范围 | 含义 | 条件 |
|------|------|------|
| 0.8~1.0 | 核心概念 | 出现在 60%+ 文档中，且每篇均为核心关键词 |
| 0.5~0.79 | 重要概念 | 出现在 30%~60% 文档中，或虽低频率但高度专精 |
| 0.2~0.49 | 次要概念 | 出现在 10%~30% 文档中，作为辅助性概念 |
| < 0.2 | 不纳入节点 | 频率太低，信息量不足 |

### 边强度（0~1）

`strength = min(共现文档数 / 总文档数 × 1.5, 1.0)`

- 共现在同一篇文档中 ≥ 3 次 → 强关联（strength 额外 +0.1）
- 跨文件夹共现 → 额外 +0.05（说明概念跨越不同子领域）
- 兜底值：若仅有同篇共现但无跨篇 → strength ≤ 0.3

## 四、输出格式

严格按以下 JSON 结构输出，确保可以被 HTML 模板正确渲染：

```json
{
  "knowledge_base": "知识库名称",
  "total_docs": 15,
  "nodes": [
    {
      "id": "node_1",
      "label": "双脑循环",
      "weight": 0.85,
      "category": "方法论",
      "description": "一段简洁的概念说明，不超过50个字",
      "docs": ["文档标题1", "文档标题2"]
    }
  ],
  "edges": [
    {
      "source": "node_1",
      "target": "node_2",
      "relation": "包含",
      "strength": 0.9
    }
  ]
}
```

### 校验规则

输出前自我检查：
1. ✅ 每个 node 的 `id` 都是 `node_X` 格式且唯一
2. ✅ 每个 node 至少关联 1 篇文档
3. ✅ 每条 edge 的 `source` 和 `target` 都能在 nodes 中找到对应
4. ✅ `weight` 和 `strength` 都是 0~1 之间的浮点数
5. ✅ nodes 数量在 20~80 之间
6. ✅ 没有孤立节点（每个 node 至少出现在 1 条 edge 中）
7. ✅ `category` 取自标准类别列表

### 标准类别列表

```
方法论, 工具, 概念, 框架, 人物, 技术, 应用场景, 理论, 模型, 策略
```
