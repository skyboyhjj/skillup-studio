# 莫比乌斯环概念地图 · 五空动画系统 · 设计文档

> 从初始创意到最终实现的全过程记录，作为下一阶段的参考。

## 一、项目背景与初心

### 1.1 灵感来源

唯识论（Yogacara）是佛教认识论最精密的体系之一，其核心机制——**种子生现行，现行熏种子**——描述了一个动态的、自我强化的认知循环。这个循环天然适合用**莫比乌斯环**来可视化：

- 莫比乌斯环只有**一个面**——正如种子与现行不是两个独立实体，而是同一个认知流的两个面向
- 环上的**扭转点**——正是"转依"（paravrtti）的发生处：有漏转为无漏
- 环的**连续性**——象征轮回的永恒运转，直到觉悟（三轮体空）打破循环

### 1.2 为什么是"五空"

唯识论的认识论中，"空"不是虚无，而是多层递进的觉知深化：

| 空 | 唯识论对应 | 可视化隐喻 |
|:--:|:---------|:---------|
| 三轮体空 | 施者空·受者空·施事空——能所双亡 | 粒子从三环融合为一个光点后消散 |
| 大圆镜智 | 阿赖耶识转依后的终极觉知——如镜照物不染 | 背景中始终存在的金色镜面光晕 |
| 觉知域 | 识的运作场域本身——非对象化的纯粹觉知 | 画布在呼吸——"活的"但不被看见 |
| 为道日损 | 道家"损之又损以至于无为"——修行是做减法 | 节点逐渐隐去，只留轮廓 |
| 色空不二 | "色即是空，空即是色"——显化即消解 | 粒子从种子层穿透三层后化为白色消失 |

**设计原则**：每个"空"占用**独立的视觉通道**（粒子、背景、画布明度、节点透明度、粒子方向），互不冲突，可独立开关。

## 二、架构演进历程

### 2.1 v1.0 — 基础引擎（~830行）

**目标**：验证莫比乌斯环3D渲染的可行性。

**核心实现**：
- 莫比乌斯环参数方程：`mobiusPoint(u, R, w, n=2)`
- 透视投影：`project(x, y, z, rotY, rotX, zoom)`
- 画家算法排序：按z深度从远到近绘制面片
- 三层环结构：种子层(R=200, yOffset=120)、现行层(R=150)、超越层(R=100)
- 三阶段动画周期（30秒）：建构(0-8s) → 流转(8-20s) → 三轮体空(20-30s)
- 粒子系统：种子层↔现行层双向流动
- 流转箭头："种子生现行"（右上升）+ "现行熏种子"（左下降）+ "转依"（中上升）
- 三轮体空动画：三金圆（施者·受者·施事）轨道运行 → 合并 → "空"字淡入

**关键决策**：
- 选择 Canvas 2D 而非 Three.js —— 零外部依赖，单文件交付
- 莫比乌斯环用**面片近似**而非参数曲线 —— 保证填充效果和画家算法兼容
- 概念数据**硬编码**在JS中 —— 快速原型验证

### 2.2 v1.0+CRUD — 数据动态化（~1208行）

**目标**：从硬编码演示升级为可管理的数据驱动工具。

**新增功能**：
- **localStorage 持久化**：`loadData()` / `saveData()` / `resetToDefault()`
- **管理面板**：左侧滑入式抽屉，包含层级Tab + 概念列表 + 添加/编辑/删除表单
- **导入/导出**：JSON文件导入 + 带日期的JSON导出
- **数据升级机制**：`upgradeData()` 自动从DEFAULT_RINGS合并缺失字段（解决localStorage旧数据覆盖新字段的问题）
- **概念ID系统**：每个概念分配唯一 `id`，支持可靠的CRUD定位
- **数据摘要**：面板底部实时显示"共23个概念 (种子层:8 / 现行层:7 / 超越层:8)"

**解决的问题**：
- 导出的是旧数据而非当前数据 → 导出前强制 `saveData()` + 从localStorage读回最新版本
- 浏览器缓存旧HTML → 添加数据版本检查 `DATA_VERSION`

### 2.3 v2.0 — 五空动画系统（~1400行）

**目标**：将哲学理念转化为五层独立的视觉动画通道。

**架构改动**：

```
v1.3:  3阶段 × 30秒
v2.0:  4阶段 × 50秒
       建构(0-8s) → 流转(8-20s) → 三轮体空(20-35s) → 为道日损(35-50s)
```

**EMPTINESS 配置系统**：

```javascript
var EMPTINESS = {
  sunyata: true,    // 空一：三轮体空
  mirror: true,     // 空二：大圆镜智
  breath: true,     // 空三：觉知域
  wane: true,       // 空四：为道日损
  ascend: true,     // 空五：色空不二
  mirrorRatio: 0.42,
  breathPeriod: 10000,
  breathAmplitude: 15,
  fadeStart: 35000,
  fadeEnd: 50000,
  fadeTarget: 0.15,
  fadeRestore: 0.5,
  ascendRatio: 0.2
};
```

全部 `false` = v1.3 纯骨架行为，完全向后兼容。

## 三、五个"空"的设计与实现

### 空一：三轮体空 · 粒子消散

**设计意图**：唯识论中三轮体空——施者空、受者空、施事空——是"能所双亡"的具体表现。在视觉上，粒子从有序流动变为减速、汇聚、融合，最终消散为一个光点。

**实现路径**：
- 第三阶段(20-35s)触发
- 粒子生成率从正常降至0（25秒时停止）
- 存活粒子被强制吸引向画布中心
- 三个金色圆环以不同速度旋转（替代v1.0的三个圆）
- 32秒开始融合为一个亮点
- 35秒完全消散，"空"字淡入

**技术要点**：
- 复用v1.0的三轮体空框架，调整时间参数适配15秒阶段
- 粒子系统的 `phase3Alpha` 控制三环的透明度渐变

### 空二：大圆镜智 · 背景镜面

**设计意图**：阿赖耶识转依后成为"大圆镜智"——如一面完美的大镜子，遍照一切而不沾不染。背景中始终存在的金色光晕就是这个"镜"的隐喻。

**设计规格**：
- 中心光晕：`rgba(255,217,61,0.15)` 径向渐变，三段式衰减（15%→8%→3%→0%）
- 外圈光环：`rgba(255,217,61,0.2)` 细线圆，1.5px
- 内圈光环：`rgba(255,217,61,0.08)` 半径50%处
- 节点倒影：每个节点下方渲染镜像，透明度8%，Y轴scale(-0.3)
- 半径自适应：`min(H, W) × 0.42`，适配不同屏幕

**调试经历**：
1. **初始值** `rgba(255,217,61,0.04)` —— 完全不可见
2. **提升到 0.12** —— 仍然不可见
3. **发现根因**：镜面渲染在莫比乌斯环**之前**，被半透明面片完全遮住
4. **修复**：将镜面渲染移到环面片**之后**、概念节点**之前**
5. **最终值** 0.15 + 外圈光环 0.2 —— 清晰可见的暖金色圆盘
6. **固定半径 300px** —— 桌面端超出画面，跨过了上下文字标注
7. **改为自适应** `mirrorRatio: 0.42` —— 移动端和桌面端均适配

### 空三：觉知域 · 画布呼吸

**设计意图**："觉知域"（Vijñapti-mātra）是唯识论的核心——万法唯识，识的运作场域本身是"活"的。画布在呼吸，但用户不应"看到"这个变化——只能"感觉到"。

**设计规格**：
- 背景色以10秒周期正弦波动
- RGB三通道同步偏移±15（最终调优值）
- 原始背景：`rgb(20,19,48)` → `rgb(10,10,26)` 径向渐变
- 呼吸时：中心 `rgb(20±15, 19±15, 48±15)` ↔ 边缘 `rgb(10±15, 10±15, 26±15)`

**调试经历**：
1. **原始设计** `breathAmplitude: 2` —— 完全不可感知（设计书原文："用户无法看到这个变化"）
2. **提升到 8** —— 仍然不可见
3. **提升到 15** —— 可以清晰看到背景在明暗交替，画布确实"活"了
4. **权衡**：从"不被看见"改为"微弱可见"——更符合展示场景的需求

### 空四：为道日损 · 节点渐隐

**设计意图**：道家"损之又损，以至于无为"与唯识论"超过而非消灭"（善种子超过恶种子，而非消灭恶种子）的共鸣。修行是做减法——节点渐隐，但不会完全消失（fadeTarget: 0.15）。

**实现路径**：
- 第四阶段(35-50s)触发
- `nodeAlphaOverride` 线性从1.0降至0.15
- 所有概念节点和标签同步渐隐
- 标语切换："施者空·受者空·施事空"淡出 → "损之又损，以至于无为"淡入至40%
- **交互恢复**：拖拽旋转时节点立即恢复至50%透明度（用户想看就能看）

**技术要点**：
- `nodeAlphaOverride` 在 `drawConceptNodes()` 开头计算，乘入每个节点的 `conceptAlpha`
- `isDragging` 标志位控制交互恢复
- 阶段末尾(>93%)触发全局淡出覆盖，为下一轮循环做准备

### 空五：色空不二 · 粒子上旋穿透

**设计意图**："色即是空，空即是色"——形式本身就是空性的显化。粒子从种子层（最底层）竖直向上穿透现行层和超越层后消失，颜色渐变象征穿过不同识层时的转化。

**实现路径**：
- 第三阶段(20-35s)触发，20%的粒子参与
- 粒子从种子层位置(startY)竖直上升到超越层位置(endY)
- X轴叠加正弦波偏移（模拟轻微摆动）
- 颜色四段渐变：`#4ECDC4`(青/种子) → `#FF6B6B`(红/现行) → `#FFD93D`(金/超越) → `#FFFFFF`(白/空)
- 穿透超越层后透明度快速衰减至0

**技术要点**：
- 粒子类型 `type: 'ascend'` 与现有双向流粒子共存
- 在 `updateAndDrawParticles()` 末尾添加 ascend 类型的渲染分支
- 渲染用径向渐变光点（5px半径），比普通粒子更大更亮

```json
{
  "rings": [
    {

      "concepts": [
        {
          "label": "阿赖耶识",
          "desc": "第八识，含藏一切种子，恒转如瀑流。",
          "docs": ["唯识三十颂", "核心概念全梳理"],
          "wuxing": "土",
          "bagua": "☷坤"
        }
      ]
    }
  ],
  "emptiness": {
    "sunyata": true, "mirror": true, "breath": true, "wane": true, "ascend": true,
    "mirrorRatio": 0.42, "breathPeriod": 10000, "breathAmplitude": 15,
    "fadeTarget": 0.15, "ascendRatio": 0.2
  }
```

### 5.2 五行八卦属性

| 属性 | 取值 | 视觉映射 |
|:-----|:-----|:-------|
| `wuxing` | 木/火/土/金/水 | Tooltip + 详情面板彩色标签 |
| `bagua` | ☰乾/☱兑/☲离/☳震/☴巽/☵坎/☶艮/☷坤 | Tooltip + 详情面板金色标签 |
| `ringLabel` | 自动取自所属ring的label | 详情面板层级标签 |

五行颜色映射：木=#2ecc71, 火=#e74c3c, 土=#f39c12, 金=#f1c40f, 水=#3498db

### 5.3 数据流

```
DEFAULT_RINGS（模板内置）
    ↓ 首次加载
localStorage（持久化）
    ↓ 每次修改
rings（内存变量）
    ↓ 渲染
projectedNodes（投影后节点列表，含wuxing/bagua/ringLabel）
    ↓ 交互
Tooltip / 详情面板
```

### 5.4 数据版本升级

```javascript

  // 从 DEFAULT_RINGS 按位置匹配合并缺失字段
```

## 九、调试记录（关键问题与解决）

### 9.1 镜面不可见

| 迭代 | 值 | 结果 |
|:----:|:---|:-----|
| 1 | `rgba(255,217,61,0.04)` | 完全不可见 |
| 2 | `rgba(255,217,61,0.12)` | 不可见 |
| 3 | 0.12 + 移到环之后渲染 | 可见！但超出画面 |
| 4 | 固定半径300px | 桌面端超出，跨过文字标注 |
| 5 | `mirrorRatio: 0.42` 自适应 | 桌面端和移动端均适配 |

**根因**：初始渲染在莫比乌斯环面片之前，被半透明面片完全遮盖。

### 9.2 画布呼吸不可见

| 迭代 | breathAmplitude | 结果 |
|:----:|:--------------:|:-----|
| 1 | 2 | 不可感知（设计书原意："看不到但感觉到"） |
| 2 | 8 | 不可见 |
| 3 | 15 | 可以看到背景明暗交替 |

**决策**：从"不被看见"改为"微弱可见"，更适合展示场景。

### 9.3 五行八卦不显示

**根因**：`projectedNodes.push()` 只传递了 `label/desc/docs/color/z`，缺少 `wuxing/bagua/ringLabel`。

**修复**：补传三个字段。

### 9.4 修改后导出仍是旧数据

**根因**：`loadData()` 从localStorage读取旧数据（无wuxing/bagua），`||` 短路使得DEFAULT_RINGS永远不被使用。

**修复**：
1. 导出前强制 `saveData()` 同步内存到localStorage
2. 导出时从localStorage读回（双重保障）
3. 新增 `upgradeData()` 自动从DEFAULT_RINGS合并缺失字段

### 9.5 概念数据ID系统

v1.0概念没有唯一ID，CRUD依赖数组索引——添加/删除后索引错乱。

**修复**：引入 `id` 字段，CRUD全部基于ID查找，`ensureIds()` 为旧数据补ID。

## 十、向后兼容策略

| 场景 | 行为 |
|:-----|:-----|
| JSON 无 `emptiness` 字段 | 五个空全部默认**开启** |
| JSON 有 `"emptiness": {}` | 按字段逐项覆盖默认值 |
| `--no-emptiness` 参数 | 全部关闭（纯骨架模式 = v1.3） |
| localStorage 有旧数据 | `upgradeData()` 自动合并新字段 |
| v1.3 生成的HTML | 无 `EMPTINESS` 变量，不受影响 |

## 十二、已知局限与下一阶段方向

### 12.1 已知局限

| 局限 | 说明 |
|:-----|:-----|
| Canvas 2D 性能上限 | 50+节点时面片绘制量增大，移动低端机可能卡顿 |
| 面片遮挡 | 莫比乌斯环面片的半透明叠加在复杂视角下可能产生视觉混乱 |
| 数据结构扁平 | 概念只有三层，不支持任意深度的树状结构 |
| 布局模式 | 八卦方位/关联/同层三种模式尚未在v2.0中完整实现 |
| 生成脚本路径 | Windows路径中的空格/中文可能导致 `build_mobius.py` 的 node 验证失败 |

### 12.2 下一阶段方向

1. **WebGL/WebGPU 渲染** —— 突破 Canvas 2D 性能上限，支持更多节点和更精细的莫比乌斯环
2. **力导向关联布局** —— 根据概念间的共现关系自动计算最优位置
3. **多主题/多数据集** —— 同一引擎支持不同知识体系的概念地图
4. **离线PWA** —— Service Worker 缓存，支持离线查看和编辑
5. **协作编辑** —— 多人同时编辑概念数据（CRDT/OT）
6. **动画录制** —— 将50秒动画循环录制为GIF/MP4用于分享
7. **AR/VR展示** —— 将莫比乌斯环概念地图放入3D空间中交互
